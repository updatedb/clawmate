# 历史会话浮层设计

> 2026-07-05

## Context

当前 Agent Panel 的历史会话是三级嵌套模式（活跃→历史列表→详情），与活跃会话共享同一个视图容器。导致 4 个问题：

1. 关闭面板再打开 → 活跃会话内容与 title/toggle 状态不一致
2. 用户习惯点 ✕ 关闭 → 面板整个关掉而不是回到活跃会话
3. 详情页的返回按钮位置不直觉，且不该显示 toggle button
4. 三级导航过深，"活跃"和"历史"应是并列而非嵌套

## 核心方案

历史会话作为**独立浮层**覆盖整个 Agent Panel：

```
Agent Panel ──────────────────────
├── Header (backend / title / [历史] [✕])
├── xterm/chat (活跃会话) ← 始终运行
├── ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─
    historyOverlay (覆盖面板)
    ├── overlayHeader
    │   ├── 列表: 历史会话 [🔍搜索] [✕]
    │   └── 详情: [←] 会话标题 [✕]
    └── overlayContent: 列表 / 详情
```

## 设计决策

| 决策 | 结论 |
|---|---|
| 历史触发入口 | Panel header 右侧 close 旁 |
| 浮层关闭后 toast 提示 | 不需要 |
| 浮层覆盖范围 | 整个面板（含 header） |
| 搜索功能 | 保留，放标题和 ✕ 之间 |
| 详情页 back/x | [←] 和 [✕] 都保留 |
| 继续会话功能 | 本次暂不实现 |
| Index/preview 复用 | 浮层逻辑全部在 agent.js，通过 domPrefix 共享 |

## 交互流程

| 操作 | 行为 |
|---|---|
| 点 Header [历史] 按钮 | 浮层从右滑入，覆盖面板 |
| 浮层 [✕] | 浮层滑出，显示底层活跃会话 |
| 点 session → 详情 | 浮层内切换为详情视图 |
| 详情 [←] | 回到浮层内列表 |
| 详情 [✕] | 直接关闭浮层，回到活跃会话 |
| 面板 [✕] | 关闭整个面板（与现在一致） |
| 关面板再开 | 永远回到活跃会话 |

## 文件变更

- `dev/static/css/style.css` — 新增浮层样式
- `dev/static/js/agent.js` — 所有浮层逻辑、重构、状态清理

**不涉及变更：** `index.html`, `app.js`, `preview.js`, 后端 API
