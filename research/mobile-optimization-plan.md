# ClawMate 手机端兼容性优化方案

> 状态：待审查 | 日期：2026-06-02

## 1. 总体目标

使 ClawMate 的 index.html（文件浏览器）和 preview.html（文件预览）在手机端可用，核心原则：

- **内容优先**：主要内容区始终可用、可滚动
- **渐进隐藏**：非核心 UI 按优先级逐步隐藏，但可通过手势/按钮恢复
- **最小集可用**：即使所有 chrome 都隐藏，核心功能（浏览/预览/反馈）不受影响

## 2. preview.html 手机端方案

### 2.1 布局变化

```
桌面端三栏                      手机端单栏
┌────┬──────────┬────┐          ┌─────────────────┐
│大纲│  内容区   │反馈│    →     │  Topbar（精简）   │
│240 │   1fr    │300 │          │  内容区（全宽）    │
├────┴──────────┴────┤          │                  │
│   BottomBar 6按钮  │          │  BottomBar（精简） │
└────────────────────┘          └─────────────────┘
                                 大纲/反馈：浮层滑入
```

### 2.2 元素分级

| 级别 | 元素 | 手机端行为 |
|------|------|-----------|
| **P0 必须** | Topbar（标题+💬反馈+🌓主题） | 精简，保留三个元素 |
| **P0 必须** | 内容显示区 | 全宽、可滚动 |
| **P0 必须** | 反馈面板（显示/隐藏） | 浮层 + 底部滑入 |
| **P0 必须** | 反馈功能（选中→备注→提交） | 简化浮层 |
| **P0 必须** | BottomBar（←返回+📑大纲+📖渲染） | 精简为 3 按钮 |
| **P1 重要** | 大纲（如文件含标题） | 点击 📑 按钮切换显示 |
| **P1 重要** | 渲染/源码切换 | 📖 按钮切换 |
| **P2 可隐藏** | "ClawMate" 品牌文字 | 手机端隐藏 |
| **P2 可隐藏** | 拷贝/导出/下载/重命名/删除 | 手机端全部隐藏 |
| **P2 可隐藏** | 音频/视频媒体面板 | 保留功能，按钮收缩 |

### 2.3 Topbar 精简

```
桌面端:  [ClawMate] [文档标题........................] [💬] [🌓]
手机端:          [文档标题.........] [💬] [🌓]
```

- 隐藏 "ClawMate" 品牌文字（节省宽度）
- `💬` 按钮：反馈面板开关
- `🌓` 按钮：主题切换
- 仅此三者，不增加其他按钮

### 2.4 BottomBar 精简

```
桌面端:  [← 返回] │ [📋 拷贝] [📥 导出] [⬇ 下载] [✏️ 重命名] │ [🗑 删除]
手机端:  [← 返回] [📑 大纲] [📖 渲染] [⬇ 下载]
```

- 仅保留 3 个按钮：返回、大纲/目录切换、渲染/源码切换
- 拷贝/导出/下载/重命名/删除 — 手机端全部隐藏
- 底部加 `safe-area-inset-bottom` 避免 iPhone 遮挡

### 2.5 大纲（左侧栏）交互

- 手机端改为 **左侧滑入浮层**（已有 @media 900px 浮层逻辑，需验证 touch 体验）
- 新增手势：从屏幕左边缘向右滑动 → 打开大纲
- 点击内容区或关闭按钮 → 关闭大纲
- 大纲内点击条目 → 跳转 + 自动关闭

### 2.6 反馈面板（右侧栏）交互

- 手机端改为 **右侧滑入浮层**（不同于桌面端的右侧栏）
- 高度全屏，从右侧滑入
- 点击内容区空白处 → 浮层消失
- 点击反馈卡片 → 弹出详情
- 手机端只显示未提交的内容项，已提交的不显示

### 2.7 反馈功能（选中→提交）适配

手机端选择文本体验与桌面端不同，需要调整：

```
桌面端:  mouseup → 浮层弹出（位置/备注/立刻执行/加入面板）
手机端:  touchend → 底部 Sheet 滑出（备注 + 立刻执行 + 加入待办）
```

- 手机端浮层改为底部 Sheet，占屏幕 40%
- 位置自动检测（不再手动输入 L{start}-{end}）
- 只保留「备注」输入框 +「加入面板」按钮
- 面板随时可「✅ 提交」（不存在满的情况）

### 2.8 渲染/源码切换

新增功能：Markdown 文件支持在渲染视图和源码视图之间切换。

```
状态: [📖 渲染]  →  点击  →  [</> 源码]
```

- Topbar 按钮 `📖` / `</>` 切换
- 渲染模式：markdown-body 格式化显示（当前默认）
- 源码模式：`<pre class="raw-text">` 显示原始 Markdown 文本
- 非 Markdown 文件不显示此按钮
- 状态通过 localStorage 持久化

### 2.9 阅读模式（沉浸式）

```
点击内容区中间  →  隐藏 topbar + bottombar，内容区全屏
再次点击       →  恢复 chrome
```

- 过渡动画 0.3s ease
- Chrome 隐藏时，在顶部保留半透明小圆点提示「点击恢复」
- 上下滑动不触发（只在 tap 时触发，move 距离 >10px 视为滑动）

### 2.10 Safe Area 适配

```css
.preview-bottombar {
  padding-bottom: max(8px, env(safe-area-inset-bottom));
  height: auto;
  min-height: 48px;
}

.preview-center-body {
  padding-bottom: max(24px, env(safe-area-inset-bottom));
}
```

### 2.11 代码块横向滚动

```css
.markdown-body pre {
  -webkit-overflow-scrolling: touch;
  overflow-x: auto;
  max-width: 100%;
}
```

### 2.12 CSS 断点

| 断点 | 目标 | 行为 |
|------|------|------|
| `> 900px` | 桌面 | 三栏全显 + 完整工具栏 |
| `≤ 900px` | 平板/手机横屏 | 单栏 + 浮层侧栏 + 精简 BottomBar |
| `≤ 480px` | 手机竖屏 | 单栏 + 最小 Topbar + 底部 Sheet 反馈 |

## 3. index.html 手机端方案

### 3.1 布局变化

```
桌面端                            手机端
┌──────┬──────────────────┐      ┌──────────────────┐
│ 侧栏  │  工具栏           │      │  Topbar（精简）    │
│      │  文件/目录卡片     │  →   │  文件/目录卡片     │
│      │  翻页             │      │  （信息瀑布）      │
└──────┴──────────────────┘      └──────────────────┘
                                    侧栏：抽屉式（已有）
```

### 3.2 元素分级

| 级别 | 元素 | 手机端行为 |
|------|------|-----------|
| **P0 必须** | Topbar（搜索+汉堡菜单） | 精简，保留 |
| **P0 必须** | Root 选择器 | 保留，可折叠收起 |
| **P0 必须** | 文件/目录卡片 | 全宽、瀑布流 |
| **P0 必须** | 内容区上下滑动 | 无限滚动/瀑布翻页 |
| **P1 重要** | 侧栏（目录树） | 抽屉式（已有） |
| **P2 可隐藏** | 卡片上的操作按钮 | 直接隐藏，不需要长按菜单 |
| **P2 可隐藏** | 多选按钮 | 隐藏 |
| **P2 可隐藏** | 画廊/列表切换 | 隐藏 |
| **P2 可隐藏** | 排序/类型过滤 | 隐藏在 ⋯ 或直接去除 |

### 3.3 文件卡片精简

```
桌面端卡片:
┌─────────────────────┐
│ [图标] 文件名.md     │
│ 1.2 KB · 昨天      │
│ [预览] [下载] [重命名] [删除] │
└─────────────────────┘

手机端卡片:
┌─────────────────────┐
│ [图标] 文件名.md     │
│ 1.2 KB · 昨天      │
└─────────────────────┘
  点击直接预览，无需操作按钮
```

- 卡片上 4 个操作按钮全部隐藏
- 不提供长按菜单（用户如需操作可切换到桌面端）
- 点击卡片直接预览

### 3.4 信息瀑布翻页

替代传统"上一页/下一页"翻页按钮：

```
滚动到底部 → 自动加载下一页（Intersection Observer）
            → 显示加载指示器
            → 追加到当前列表末尾
```

实现方式：
- `IntersectionObserver` 监听底部 sentinel 元素
- 进入视口 → 自动请求 `?page=N+1`
- 占位 sentinel：`<div id="scrollSentinel" style="height:1px">`
- 加载中显示底部 spinner
- 到最后一页时 sentinel 移除

### 3.5 工具栏精简

手机端工具栏全部隐藏，不需要排序/类型/视图切换。

### 3.6 搜索栏

- 已有适配（保持）
- 搜索时顶部固定，内容区下方滚动

### 3.7 批量操作栏

- 多选按钮隐藏 → 批量操作栏不出现
- 如果未来需要，通过长按进入选择模式

## 5. CSS 变量预案

新增/调整的 CSS 变量：

```css
:root {
  --safe-bottom: env(safe-area-inset-bottom, 0px);
  --mobile-sheet-height: 60vh;
  --mobile-sheet-radius: 16px;
}
```

## 6. 不处理项

| 项目 | 理由 |
|------|------|
| ONLYOFFICE 编辑 | 手机端 Office 编辑体验差，保持只读预览 |
| 视频/音频面板 | 已有原生播放器，无需额外适配 |
| PDF 预览 | pdf.js 已有自带移动端适配 |
| 批量操作 | 多选按钮已隐藏，批量操作栏不会出现 |
| PWA / Service Worker | 本次不涉及离线缓存 |

---

## 7. 风险点

| 风险 | 缓解 |
|------|------|
| IntersectionObserver 加载抖动 | debounce 500ms，加载指示器预留 60px 高度 |
| 反馈底部 Sheet 与键盘冲突 | Sheet 高度自适应，键盘弹出时 Sheet 上移 |
| Safe area 在 Android 上无效 | 用 `max()` 降级到固定 padding |
